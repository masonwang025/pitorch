# ops/core — CPU reference ops for LLM inference

All ops operate on flat `float*` arrays with no allocations. These are the CPU reference implementations used to validate GPU kernels.

## Software math (`pt_math.h`)

Bare-metal Pi has no libm. Polynomial approximations with range reduction, all < 1e-5 relative error:

- `pt_expf` — degree-5 Taylor after `x = n*ln2 + r` reduction, 2^n via IEEE754 exponent
- `pt_sinf` — degree-9 odd polynomial on [-pi/2, pi/2], Cody-Waite reduction
- `pt_cosf` — degree-8 even polynomial, same reduction, sign flip for outer quadrants
- `pt_sqrtf` — bit-trick initial guess + 3 Newton-Raphson iterations

## Ops (`pt_ops.h`)

| Op | Signature | Notes |
|----|-----------|-------|
| `rmsnorm` | `(o, x, w, dim)` | RMS normalize, scale by learned weight |
| `softmax` | `(x, size)` | in-place, numerically stable (max subtraction) |
| `silu` | `(x, size)` | in-place, x * sigmoid(x) |
| `rope` | `(q, k, dim, head_dim, pos)` | rotary position encoding, computes freqs via `pt_expf` |
| `vec_add` | `(o, a, b, size)` | element-wise add (residual connections) |
| `vec_mul` | `(o, a, b, size)` | element-wise multiply (SwiGLU gate) |
| `embedding_lookup` | `(o, table, dim, token)` | memcpy of one row |
| `argmax` | `(x, n) → int` | greedy sampling |
| `smatvec_cpu` | `(W, x, y, out_dim, in_dim)` | y = W @ x, CPU reference matvec |
